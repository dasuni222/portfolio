Directed Graph Cycle Detection

This coursework focuses on determining whether a directed graph is cyclic or acyclic using algorithmic techniques. The project implements and analyzes two approaches: the Sink Elimination Algorithm and DFS-based cycle detection, using benchmark datasets to compare behavior and performance.

Overview

A directed graph is cyclic if a path returns to its starting vertex. In this project, both algorithms were tested on benchmark examples to confirm whether the graph contained a cycle and to compare their efficiency for different graph sizes.

Algorithms Used

1. Sink Elimination Algorithm: repeatedly removes sink vertices and their incoming edges until all vertices are removed or no sink remains, which indicates a cycle.

2. DFS-Based Cycle Detection: traverses each vertex deeply, marking visited nodes and detecting back edges to identify cycles.

Benchmark Examples

Acyclic Graph: 0 → 1 → 2 → 3. Both algorithms confirm the graph is acyclic.

Cyclic Graph: 0 → 1 → 2 → 0. Sink elimination fails to remove all vertices, while DFS detects a back edge, confirming the graph is cyclic.

Performance Analysis

Sink Elimination runs in O(V × E) time with O(V + E) space, while DFS cycle detection runs in O(V + E) time with O(V) space. The findings show that DFS is more efficient and scalable for larger graphs, while sink elimination is simpler and effective for smaller cases.
